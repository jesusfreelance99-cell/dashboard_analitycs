# Configuración de Firebase - Trevo Analytics

## Información del Proyecto

```
Nombre: Trevo IA
ID: trevo-ia
Número: 887325226758
Organización: ghc-digital.com
```

## Paso 1: Descargar archivo de configuración

1. Ve a [Firebase Console](https://console.firebase.google.com/)
2. Selecciona el proyecto **Trevo IA**
3. Ve a **Configuración del proyecto** (⚙️)
4. En la pestaña **General**, desplázate hasta **Tus aplicaciones**
5. Busca la aplicación Web (si no existe, agrégala)
6. Descarga el archivo de configuración JSON

## Paso 2: Crear archivo de configuración

Crea el archivo: `lib/firebase_options.dart`

```dart
// This file was automatically generated by flutterfire_cli.
// https://github.com/invertase/flutterfire_cli

import 'package:firebase_core/firebase_core.dart' show FirebaseOptions;
import 'package:flutter_dotenv/flutter_dotenv.dart';

class DefaultFirebaseOptions {
  static FirebaseOptions get currentPlatform {
    return web;
  }

  static const FirebaseOptions web = FirebaseOptions(
    apiKey: 'YOUR_API_KEY',
    appId: 'YOUR_APP_ID',
    messagingSenderId: 'YOUR_MESSAGING_SENDER_ID',
    projectId: 'trevo-ia',
    authDomain: 'trevo-ia.firebaseapp.com',
    databaseURL: 'https://trevo-ia.firebaseio.com',
    storageBucket: 'trevo-ia.appspot.com',
    measurementId: 'YOUR_MEASUREMENT_ID',
  );
}
```

**Reemplaza los valores con los del archivo de configuración descargado.**

## Paso 3: Inicializar Firebase en main.dart

```dart
import 'firebase_options.dart';

void main() async {
  WidgetsFlutterBinding.ensureInitialized();
  
  // Inicializar Firebase
  await Firebase.initializeApp(
    options: DefaultFirebaseOptions.currentPlatform,
  );
  
  await EasyLocalization.ensureInitialized();

  runApp(/* ... */);
}
```

## Paso 4: Configurar Autenticación Firebase

### Habilitar Email/Password

1. Ve a **Authentication** en Firebase Console
2. Pestaña **Sign-in method**
3. Habilita **Email/Password**

### Habilitar Google Sign-In

1. Ve a **Authentication** → **Sign-in method**
2. Habilita **Google**
3. Configura el email de soporte del proyecto

### Configurar OAuth para Web

Para que Google Sign-In funcione en web:

1. Ve a [Google Cloud Console](https://console.cloud.google.com/)
2. Selecciona el proyecto **trevo-ia**
3. Ve a **APIs y servicios** → **Credenciales**
4. Crea una credencial **OAuth 2.0 Client ID** (tipo Aplicación web)
5. Agrega los orígenes autorizados:
   - `http://localhost:5000`
   - `http://localhost:5173`
   - `https://trevo-ia.firebaseapp.com`
   - (Tu dominio de producción)
6. Copia el **Client ID** en `google_auth_service.dart` si es necesario

## Paso 5: Estructura de Firestore

Crea la colección `users_dashboard` con estructura:

```json
{
  "users_dashboard": {
    "user_id": {
      "email": "admin@trevo.com",
      "displayName": "Admin Name",
      "rol": "admin",
      "createdAt": "2024-06-06",
      "lastLogin": "2024-06-06"
    }
  }
}
```

**Campos requeridos:**
- `email` (String) - Email del usuario
- `rol` (String) - Debe ser "admin" para acceder
- `displayName` (String) - Nombre del usuario
- `createdAt` (Timestamp) - Fecha de creación
- `lastLogin` (Timestamp) - Último acceso

## Errores Comunes

### "user-not-found en Firestore"
- ✅ Solución: Crea el usuario en la colección `users_dashboard` con `rol: "admin"`

### "Acceso denegado: rol no es admin"
- ✅ Solución: Verifica que el campo `rol` sea exactamente `"admin"` (case-sensitive)

### Google Sign-In no funciona en web
- ✅ Solución: Configura OAuth Client ID en Google Cloud Console
- ✅ Solución: Agrega los orígenes autorizados

### "Network error" en producción
- ✅ Solución: Verifica que los dominios estén en Firebase Console y Google Cloud

```dart

  appId: dotenv.env['FIREBASE_APP_ID']!,
  // ... resto de campos
);
```
